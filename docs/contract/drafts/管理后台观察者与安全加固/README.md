# 管理后台观察者与安全加固契约草案

本目录保存 M1/M2/M3 的阶段性契约草案。草案用于记录已经实现并有验证证据、但尚未满足正式契约同步闸门的事实，不代表 `docs/contract.md` 已生效。

## 文件与同步规则

| 草案 | 覆盖范围 | 生成时点 | 正式契约同步时点 |
|---|---|---|---|
| `M1_契约草案.md` | STEP-001～017 的阶段 A 安全事实 | M1 验收完成时 | STEP-017 人工确认后 |
| `M2_契约草案.md` | STEP-018～029 的 observer 后端和公共前端底座事实 | M2 验收完成时 | 不单独同步，等待 STEP-040 |
| `M3_契约草案.md` | STEP-030～040 的页面闭环和最终五角色事实 | M3 验收完成时 | STEP-040 全部验收、独立部署及人工确认后 |

## 草案条目格式

每条契约事实使用以下字段：

- 条目 ID：`M{里程碑}-STEP-{编号}-{序号}`；
- 来源 STEP；
- 接口、数据结构或前端能力；
- 前置条件和权限边界；
- 请求、响应或状态变化；
- 明确排除范围；
- 实现文件；
- 自动化或人工验证证据；
- 验证日期；
- 正式契约同步状态。

只有相应 STEP 的五项完成标志全部满足，条目才可标为“已验证”。尚未通过、缺少真实环境证据或等待人工确认的内容必须标为“草稿”或“阻塞”，不得写成正式事实。
