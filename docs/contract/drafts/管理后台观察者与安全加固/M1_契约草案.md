## 本阶段新增/变更摘要

- 后台 JWT 密钥必须显式配置，配置读取与应用启动执行同一非法值守卫。
- `admin_users` 新增会话版本；Admin JWT 携带并按数据库实时版本、账号状态校验。
- 登录失败计数使用 MySQL 行锁；第五次错误原子锁定并撤销旧会话；四类登录失败统一对外响应。
- 自助改密、登出、重置密码和实际角色变化撤销旧会话；备注、相同角色和解锁不误递增版本。
- life-config 发布服务端强制精确 `CONFIRM`，现有 7 个前端入口显式传值。
- 操作日志写入与读取、系统日志读取统一使用共享、递归、幂等、失败关闭的凭据脱敏工具。
- 本草案不新增 `observer`，角色体系继续保持 `super_admin`、`ops_admin`、`ai_trainer`、`tech_ops`。

## 环境与启动契约

### M1-STEP-001-01 · `ADMIN_JWT_SECRET`

- 必须显式提供后台 JWT 密钥。
- 缺失、空值、纯空白、`admin_secret_change_me`、`your_admin_jwt_secret_here` 均导致配置读取或应用启动失败。
- 合法自定义值可继续使用；本阶段不新增长度、复杂度、轮换或强制换密规则。
- 用户端 `JWT_SECRET` 不受本规则影响。

## Admin JWT 与会话契约

### M1-STEP-003-01～M1-STEP-008-01

- 新 Admin JWT payload 包含整数 `token_version`；`sub`、`role`、`type=admin`、签名和过期规则保持原契约。
- 统一鉴权以数据库账号为最终事实，依次验证账号存在、启用、未锁定和 Token 版本匹配。
- 历史无版本 Token、非整数版本、版本不匹配以及无效账号状态统一返回 HTTP 401。
- 下列事件成功后 `token_version` 精确递增一次：第五次错密锁定、自助改密、登出、超级管理员重置他人密码、他人角色实际变化。
- 仅修改备注、提交相同角色、手动解锁不递增；解锁不恢复锁定前 Token。
- 登出是账号级全会话撤销，不实现单设备 Token 黑名单。

## 登录安全契约

### M1-STEP-004-01 · 行锁与第五次锁定

- 已存在账号的登录查询使用 `SELECT ... FOR UPDATE`。
- 锁定检查、密码校验、失败计数、第五次锁定、成功清零在同一事务中完成。
- 第五次错误后固定为 `login_fail_count=5`、`is_locked=true`，且仅在锁定事件递增一次 `token_version`；锁定后继续尝试不再改变状态。

### M1-STEP-005-01 · 统一失败响应

不存在账号、密码错误、锁定、停用四类失败均返回：

```json
{"code": 20001, "data": null, "message": "账号或密码错误"}
```

HTTP 状态为 200；不存在账号执行固定伪 bcrypt；真实原因只进入不含提交密码、Token 或 Secret 的服务端安全日志。

## 管理端 API

### M1-STEP-009-01 / M1-STEP-010-01 · `POST /api/admin/life-config/publish`

- 请求体新增可缺省的 `confirm_text`。
- 进入发布逻辑前必须满足 `confirm_text == "CONFIRM"`；缺失、`null`、空值、纯空白或其他文本返回业务码 `20021`。
- CONFIRM 不替代既有角色鉴权，也不是二次认证。
- 现有 7 个前端调用均在共享 `showConfirmInput` 回调中显式发送 `confirm_text: "CONFIRM"`；取消或错误文本不调用发布。
- 不改变草稿、生效版本、历史、回滚、Redis 发布流程或数据库结构。

## 日志凭据脱敏契约

### M1-STEP-011-01 · 共享工具

- `redact_credentials()` 无状态、幂等，命中值统一替换为 `[REDACTED]`。
- dict/list 递归；字符串先尝试 JSON 解析，非 JSON 仅精确匹配已知凭据赋值、Authorization 和具有凭据形态的 Bearer。
- 大小写不敏感匹配 `api_key`、`secret_key`、`api_secret`、`access_key_secret`、`access_token`、`refresh_token`、`authorization`、`password`、`pepper`、`private_key` 及语义明确的 `_secret`、`_token`。
- Prompt、普通配置、对话、记忆、描述、版本与 `max_tokens` 不误脱敏；单字段异常失败关闭但不让业务失败。

### M1-STEP-012-01 · 操作日志

- `log_operation` 写入 `target_description`、`before_value`、`after_value` 前脱敏。
- 操作日志列表、详情与 Excel 导出返回前再次调用同一工具，遮蔽历史遗留明文。
- 不批量改写历史数据库行；具备数据库直接访问权限者仍可能看到历史明文。

### M1-STEP-013-01 · 系统日志

- 系统日志列表分页结果和 Excel 导出行返回前调用同一共享工具。
- 所有获准管理员角色得到一致的安全读取结果；日志收集、筛选、排序和日期范围不变。
