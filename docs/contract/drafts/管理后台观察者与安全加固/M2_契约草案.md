## 本阶段新增/变更摘要

- 管理员账号角色值扩展 `observer`，展示名为“观察者”；账号管理继续仅限 `super_admin`。
- 后端在完整身份校验后对 observer 的 POST/PUT/PATCH/DELETE 默认拒绝，仅精确放行登出和自助改密。
- 匿名 OPTIONS 只作为 CORS 预检；操作日志、数据报表、系统日志三个导出端点显式拒绝 observer。
- 已批准业务模块拆分读写角色集合，observer 可读但不能写、测试、生成、发布、回滚、导入、导出或管理账号。
- 第三方凭据及用户 Open API Key 对 observer 只返回配置状态，不返回任何可推断片段。
- 前端提供 observer 菜单、Header、公共只读助手、动态节点观察及 `adminRequest()` 非读取请求兜底。
- 本阶段不修改正式 observer/RBAC 契约，不声明 35 个页面已经完成只读闭环。

## 角色与账号契约

### M2-STEP-018-01 · 角色值与账号管理边界

- `observer` 是管理员账号合法角色值，展示名为“观察者”。
- 创建、编辑、列表展示支持该角色；数据库字段结构不变。
- 账号列表、创建、编辑、删除、重置密码和解锁继续仅允许 `super_admin`。
- 不新增管理员启用/停用、统一角色注册中心或账号审批流程。

## 统一权限边界

### M2-STEP-019-01 · 方法级写总闸

- 统一鉴权先完成 JWT 类型、签名、过期、`token_version`、账号存在/启用/未锁定及数据库实时角色校验。
- 随后对 observer 的 POST/PUT/PATCH/DELETE 默认返回 HTTP 403，端点业务函数不执行。
- 唯一例外为 `POST /api/admin/auth/logout` 和 `POST /api/admin/auth/change-password`；相似路径、错误方法不放行。
- GET/HEAD 仍须通过各路由原有角色依赖，前端兜底不替代后端安全边界。

### M2-STEP-020-01 · CORS OPTIONS

- 带合法预检头的匿名 OPTIONS 由 CORS 中间件返回纯文本 `OK` 和 CORS 响应头，不返回业务数据、不进入业务函数。
- 匿名业务 GET、显式 HEAD 和写请求仍进入统一 Admin 鉴权。

### M2-STEP-021-01 · 内建导出

以下三个端点保留原角色依赖，并额外挂载方法无关的 observer 专用拒绝依赖：

- 操作日志导出；
- 数据报表导出；
- 系统日志导出。

observer 不获得文件响应；未来新增 export/download 路由必须通过路由清单门禁。

## 业务读取与写入契约

### M2-STEP-022-01～M2-STEP-026-01

- 用户、统计与脱敏操作日志：observer 可读用户列表/详情、对话、情绪轮次、日记、现有统计及操作日志列表/详情；写与导出不开放。
- AI 配置与测试：人格、Prompt、只读对话流 Prompt、测试用例、安全规则的 GET/HEAD 可读；保存、测试、发布、回滚、导入和删除不开放。
- 记忆、向量、知识等模块：记忆、非敏感向量数值、知识、Agent、关系/日记、情绪、世界状态及用户向量记忆可读；连接测试、生成、重置和增删改不开放。
- 系统与第三方：系统状态、第三方运行状态和脱敏系统日志可读；第三方配置保存、连接测试和日志导出不开放。
- 生活流：life-config、生活计划、朋友圈内容、评论、感知队列和世界观现有 GET 可读；草稿、发布、回滚、生成、显隐、重试、删除和重置不开放。
- 原四角色的既有合法权限保持不变。

## 敏感凭据读取契约

### M2-STEP-027-01 · 只返回配置状态

- 第三方状态仅对带凭据的服务返回布尔 `credential_configured`。
- 用户 Open API Key 对 observer 仅返回布尔 `enabled`。
- 响应不得包含凭据原文、prefix/suffix、首尾或掩码片段、哈希及可推断凭据的时间元数据。
- 其他获准角色的既有响应契约保持不变；所有保存、生成和连接测试继续拒绝 observer。

## 全量路由审计契约

### M2-STEP-028-01

- 当前自动枚举 159 个 `/api/admin/*` 路由：69 个 GET/HEAD、90 个写方法、3 个 export/download。
- 除唯一匿名登录 POST 外，全部路由具有递归统一 Admin 鉴权依赖。
- 所有写路由进入 observer 方法总闸，三个导出路由命中专用拒绝依赖。
- GET/HEAD 处理器不得执行数据库或业务写调用；仅系统状态和第三方状态允许有限 TTL 缓存回填。

## 公共前端契约

### M2-STEP-029-01 · 菜单、Header、只读助手与请求兜底

- `isObserver()` 统一判断当前角色；observer 菜单复用完整业务读取导航但排除账号管理，Header 显示“观察者”。
- `data-write-action` 是后续页面改造的统一写操作标记。
- `applyObserverReadOnly()` 只处理带标记节点：写按钮/链接隐藏，展示型 input/textarea 只读，select、checkbox/radio 开关和 file 控件禁用。
- `MutationObserver` 对首屏后插入的带标记节点重复应用同一规则；不粗暴禁用未标记按钮或全部表单。
- `adminRequest()` 对 observer 仅允许 GET/HEAD，以及精确的登出、自助改密 POST；其他请求在发出前返回 `null` 并提示只读。
- 搜索、筛选、分页、Tab、详情、复制和纯 GET 刷新必须保持可用；页面专项标记与动态弹窗闭环由 STEP-030～038 完成。
