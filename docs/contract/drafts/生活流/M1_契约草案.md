## 用户端 API

前缀 `/api/feed`，**全部 JWT 鉴权**。响应格式：`{"code":0,"data":{},"message":"success"}`。

### GET `/api/feed/list`

| 参数 | 说明 |
|------|------|
| `cursor` | 可选；上一页最后一条 `scheduled_publish_time`（ISO） |
| `size` | 默认 20，上限 50 |

**查询条件**：`scheduled_publish_time<=feed_now()` · `is_visible=1` · `generation_status='ready'` · 可见范围（`feed_history_visible_range`：`7d`/`30d`/`180d`/`all`，窗口边界亦相对 `feed_now()`）。

> **时区**：响应中 `scheduled_publish_time` 为北京墙钟 ISO（无 `Z` 后缀，语义=上海本地）；H5 按墙钟字符串展示，不按 UTC 解析。

**响应 `data`**：

```json
{
  "posts": [{
    "id": 1,
    "content_text": "...",
    "hashtags": ["#日常"],
    "image_urls": ["https://cdn.../01.webp"],
    "scheduled_publish_time": "2026-06-15T10:30:00",
    "emotion": "平静",
    "city": "杭州",
    "display_likes": 35,
    "user_liked": false,
    "display_comments": 12,
    "comments": [{
      "id": 10,
      "content": "用户评论",
      "reply_to_lxm": false,
      "lxm_reply": "林小梦回复",
      "lxm_reply_at": "2026-06-15T10:35:00",
      "lxm_reply_read_at": null
    }]
  }],
  "next_cursor": "2026-06-14T15:00:00"
}
```

- 评论**仅返回当前用户**在该帖下的记录（私有）。
- 列表命中且 `actual_publish_time IS NULL` 的帖子批量懒惰写回。
- **`city`（2026-07-11）**：来自 `feed_post.city`（场景城市，发布时已写入）；空串表示无城市；**无 DB 迁移**。第一版仅出参 + H5 展示，不含天气字段。
- **`reply_to_lxm`（2026-07-12）**：`boolean`；缺省/历史数据等价 `false`。
- **`display_comments`（2026-07-12）**：`base_comments × comment_multiplier + len(本帖当前用户可见 comments)`；**非** DB 列。历史帖 `base_comments=0, comment_multiplier=1` → 等于真实可见条数；角标可大于评论区可见条数（与点赞假数同模式）。发评后 H5 本地角标 `+1`（与点赞乐观更新同思路）。

### POST `/api/feed/enter`

写 `users.last_feed_entered_at=feed_now()`；返回最近一条未读 LXM 回复的评论 ID。

```json
{"anchor_comment_id": 10}
```

### GET `/api/feed/badge`

```json
{"has_new": true, "new_post_count": 8, "unread_reply_count": 3}
```

- `has_new`：存在已到点可见帖且（`scheduled_publish_time > last_feed_entered_at`，或从未进入）；计数与 list 对齐历史可见窗。
- `new_post_count`：同上口径的**新帖条数**；**从未进入**（`last_feed_entered_at IS NULL`）时固定为 `0`（避免首页底栏展示全量历史 N；此时若有可见帖仍 `has_new=true`，前端底栏展示「快来看看」）。
- `unread_reply_count`：当前用户未读 LXM 评论回复数。
- 首页 UI（2026-07-11）：未读回复 → 顶胶囊 `NEW · N条新回复`；新帖数 → 底栏 `N 条新动态`；二者可同时展示（不再互斥红点/数字）。
- 点击：`unread_reply_count > 0` → `?focus=unread_reply`，否则进列表顶。

  "display_nickname": "林小梦"

### POST `/api/feed/{post_id}/like`

幂等点赞；真实新增时 `real_likes+1`；调用 `like_aware_service.on_like_hook`（M2 起为真实入队；**M1 曾为 stub**）。

**事务时序（2026-07-11 · TB-LF-009）**：服务层须先 `flush` + **`commit` 点赞事务**（释放 `feed_post` 行锁），再 `await on_like_hook`。钩子另开 session 写 `agent_aware_queue`（FK→`feed_post`）；若在未提交事务内调用，易 `Lock wait timeout`，入队失败被吞而点赞仍成功。细则与特殊档计数见 `M2_契约草案.md`。

响应：`{"user_liked": true, "display_likes": 36}`

### DELETE `/api/feed/{post_id}/like`

幂等取消；`real_likes-1`（`AND real_likes>0`）；**不撤回**已触发感知 IM。

### POST `/api/feed/{post_id}/comments`

Body：`{"content": "...", "reply_to_lxm": false}` · `content` 长度 1~200 · 同帖 30s 频控 · 内容安全。

| 字段 | 必填 | 说明 |
|------|------|------|
| `content` | 是 | 评论文本 |
| `reply_to_lxm` | 否 | 默认 `false`；`true`=用户点击林小梦回复行后发出，落库 `reply_to_lxm=1` |

响应：`{"comment_id": 10, "created_at": "...", "gen_status": "pending", "reply_to_lxm": false}`

**首次评论**：原子 `UPDATE relationship SET has_ever_commented_feed=1 WHERE ...=0` 成功 → `due_at=utcnow()+30s`；否则按关系档随机延迟（`due_at`/`created_at` 为 UTC；到点可见性校验仍用 `feed_now()`）。

**连续回复（2026-07-12）**：服务端仍为「同帖多条独立评论、各条 LLM-05 必回」；**无**对话链上下文注入；`reply_to_lxm` **仅展示标记**。

### 同帖多图与限速（2026-07-10 · TB-LF-008）

| 项 | 约定 |
|----|------|
| 张数来源 | `feed_image_count_*_weight` 抽签；单次 Liblib 任务 `generateParams.imgCount` **写死 1**（N 张 = N 次提交） |
| 类型 | 整帖只抽一次 `image_type`，多图共用 |
| `count == 1` | 正向 prompt **不**追加变体；`seed = -1` |
| `count ≥ 2` | 在主 prompt 末尾按 `seq` 追加构图短句（selfie / 非 selfie 两套代码常量）；每张独立随机 `seed`；打日志 |
| 提交 QPS | ≤ 1/s（`liblib_client` 提交后 sleep 1s） |
| 进行中任务并发 | **1**（`concurrency_sem`；账号侧「进行中任务上限」实测约 1，并行会报错） |
| 失败降级 | 部分失败保留已成功 URL；全失败 → 空数组纯文字帖 |

| 项 | 值 | 说明 |
|----|----|------|
| 单次请求超时（全局默认） | **45s** | `DEEPSEEK_DEFAULT_TIMEOUT`（`backend/utils/deepseek_client.py`） |
