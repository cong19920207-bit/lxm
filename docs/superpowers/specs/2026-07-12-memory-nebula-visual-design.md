# 记忆星云表现层优化设计

## 目标

在不改变记忆数据来源、关系计算方式和页面核心交互的前提下，增强记忆星球关系网络的可读性，降低顶部信息的突兀感，并统一页面周边控件的沉浸式玻璃质感。

成功标准：

- 默认状态能明显看出星球之间存在关系网络，同时不会形成难以辨认的“蜘蛛网”。
- 点击任意记忆星球后，用户能立即分辨选中星球、核心路径、关联路径和非关联内容。
- 顶部标题、数量信息和返回按钮与星云画面融合，不遮挡主要星体。
- 保持现有拖动、缩放、自动旋转、记忆卡片、回到中心和减少动态效果逻辑。

## 视觉方案

采用“能量脉络网”方案。连接线使用亮芯、柔光和体积线三层结构；默认态展示较完整的关系网络，选中态通过颜色、透明度、宽度、动画和场景降亮形成明确层级。

### 默认关系网络

- 根据屏幕和记忆数量动态展示关系线，移动端最多显示 14 条，较宽屏最多显示 18 条。
- 关系线优先级为：同分类关系、核心引力关系、空间最近邻关系。
- 每条线包含：细亮芯线、低透明柔光层、保证移动端可见的体积线。
- 同分类关系使用冷紫白色；核心引力关系从暖金色过渡到淡紫白；最近邻关系透明度最低。
- 根据连线在相机中的远近调整透明度：近处清晰、远处衰减，但最低仍保留可辨认轮廓。
- 默认线保持缓慢、低幅度呼吸；仅选择一条关系线播放稀疏流光，避免所有线同时运动。

### 星球选中状态

- 用户点击记忆星球后，选中星球放大约 20%，增加柔和外环和一次短促点亮反馈。
- 从记忆核心到选中星球的主路径使用金色到星球主题色的渐变，亮芯最清晰，并播放周期性流光。
- 选中星球与最多两个关联星球之间显示次级路径，宽度和亮度低于主路径。
- 非相关默认线降至原透明度的约 15%；非相关星球降低亮度并轻微缩小。
- 再次选择其他星球时平滑切换路径；关闭记忆卡片后恢复默认网络。
- 系统开启“减少动态效果”时保留静态高亮和层级变化，关闭呼吸、流光和扩散环动画。

## 顶部信息结构

顶部中心改为轻量双行信息：

- 主标题：「她记得的你」
- 副标题：动态显示「N 颗记忆星体」

主标题使用较小字号和中等字重，副标题使用低透明度小字号。右上角不再重复展示数量，改为纯信息按钮；点击后继续展示现有记忆说明。

顶部区域增加从深色到透明的渐隐遮罩，仅用于保证文字和控件可读，不形成实体导航栏。

## 周边控件

### 返回按钮

- 使用深紫黑半透明圆形毛玻璃。
- 加入一像素半透明描边、轻微内高光和柔和投影。
- 按下时缩放至约 94% 并提高背景亮度；保持现有返回行为和 44 像素触控区域。

### 信息按钮

- 采用与返回按钮一致的圆形玻璃样式，保留信息图标。
- 视觉权重低于返回按钮，不再承载数量文案。

### 底部操作提示

- 将「拖动可探索星云」升级为半透明玻璃胶囊。
- 文案改为「拖动探索 · 点击查看记忆」，配合现有手势图标。
- 用户开始交互或打开记忆卡片时淡出，空闲后恢复。
- “回到中心”仅在视角明显偏移时出现，沿用相同玻璃语言。

## 实现边界

- `frontend/static/js/memory-connection-layer.js`：调整默认连接数量、关系类型视觉参数、深度衰减、默认流光和选中态层级。
- `frontend/static/js/memory-nebula.js`：补充选中外环、顶部动态副标题、交互提示显隐，并维持现有卡片与手势逻辑。
- `frontend/pages/memory.html`：调整顶部和底部 DOM 文案与样式，增加渐隐遮罩和统一玻璃控件状态。
- 不修改后端接口、记忆关系计算语义、星球素材及记忆卡片内容结构。
- 不引入新的第三方依赖。

## 验证要求

- 在 390×844 和 430×932 移动端视口检查默认态、选中态、卡片展开态和偏离中心态。
- 验证默认关系线数量上限、关系类型优先级和关闭卡片后的恢复状态。
- 验证选择不同颜色类别的星球时，主路径渐变和关联路径可辨认。
- 验证返回、信息说明、拖动、缩放、点击星球、关闭卡片和回到中心没有交互回归。
- 验证 `prefers-reduced-motion: reduce` 下没有持续呼吸、流光或扩散动画。
- 运行现有 H5 静态契约测试，并检查浏览器控制台无新增错误。
