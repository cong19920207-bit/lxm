## 三、顶部区域（Header）

### 3.1 元素构成

| 位置 | 元素 | 说明 |
|---|---|---|
| 左上角 | 返回按钮 | 返回上一页（主页或消息页） |
| 右上角 | 消息入口 icon | 跳转 IM 对话页 **`/pages/chat.html`**，展示林小梦主动 IM 消息未读提示（与主页聊天入口共用 `/api/agent/unread-count`，见 PRD 5.3） |

- Header 不吸顶，随 Feed 整体上滑
- 返回按钮与消息 icon 悬浮在背景图上方，始终可点击

## 四、Feed 内容列表

### 4.1 单条动态布局（v1.1 移除天气 icon）

```
┌──────────────────────────────────┐
│  11月              正文文字内容... │
│  23日                继续正文...   │
│                   🖼️ 🖼️ 🖼️      │
│                   🖼️ 🖼️ 🖼️      │
│  刚刚                    👍  💬  │
│  ────────────────────────────── │
│  林小梦：[高亮] 评论内容...        │
│  我：评论内容...                   │
└──────────────────────────────────┘
```

**左侧列（固定宽度）：**
- 第一行：月份，如 `11月`
- 第二行：日期，如 `23日`
- 第三行（右侧内容底部对齐）：相对时间，如 `刚刚`

**右侧列（剩余宽度）：**
- 正文文字
- 图片区域
- 底部行：左侧无内容（时间在左侧列）/ 右侧 👍 icon + 💬 icon
- 评论区（有评论时显示）

---

### 4.2 图片展示规则（v1.1 更新上限）

| 图片数量 | 展示方式 |
|---|---|
| 1 张 | 全宽显示 |
| 2 张 | 并排两列 |
| 3 - 4 张 | 3 列网格排列 |

> 说明：LIFE001 图片生成上限为 4 张，不会产出 5-9 张内容，故前端展示规则按 4 张上限设计（v1.0 原规则覆盖至 9 张，本轮收紧）。

**图片点击行为：**
- 点击图片 → 全屏预览
- 全屏预览支持左右滑动切换多图
- 支持双指缩放

> ⚠️ **技术债 TD-001**：H5 全屏预览的左右滑动手势与页面返回手势存在冲突风险，需在实现阶段处理手势优先级；此逻辑已为未来 APP 端复用预留。

### 4.3 时间显示规则（微信同款）

| 时间范围 | 显示文案 |
|---|---|
| 发布时间 < 1 分钟 | 刚刚 |
| 1 分钟 ≤ t < 60 分钟 | X 分钟前 |
| 1 小时 ≤ t < 24 小时 | X 小时前 |
| 昨天 | 昨天 |
| 2 天及以上 | X 天前 |

> **时区（2026-07-10）**：`scheduled_publish_time` 与相对时间日界均为 **北京时间（Asia/Shanghai）**；实现侧按墙钟字符串解析，不依赖设备本地时区。当前 H5 实现文案为「当天 HH:mm / 昨天 / 周X / MM-DD HH:mm」，产品表上「刚刚/X 分钟前」为规范目标，二者均须基于北京墙钟。

### 4.4 互动栏

**布局：**
- 底部行左侧：相对时间（在左侧列）
- 底部行右侧：👍 icon + 💬 icon

**点赞交互：**
- 单击 👍 → 点赞，icon 高亮
- 再次单击 → 取消点赞，icon 恢复默认
- 点赞数量展示规则见 PRD 6.1.1，此处不重复定义
- 当前用户是否已赞由 `feed_like` 表判定（见 PRD 11.4）
- 点赞后是否触发林小梦 IM 私信由后台异步判定（特殊窗口 100% / 常规 30% 等），**前端无即时提示**；规则见 PRD **6.1.3**；与已读 IM 互斥见 PRD **第七章**

**评论角标数值（2026-07-12 增量，对齐点赞假数）：**
- 展示评论数 = `base_comments × comment_multiplier + 当前用户可见评论条数`
- `base_comments` / `comment_multiplier`：发帖时随机确定（范围与点赞 `feed_base_likes_*` / `feed_like_multiplier_*` 相同），固定不变；**仅放大 base，不放大真实评论数**
- 历史帖默认 `0 × 1`（不回填），角标等于真实可见条数
- API 字段：`display_comments`（list 组装，非 DB 列）；H5 发评成功后角标本地 `+1`
- 评论区内容仍为当前用户私有（见 4.5）；角标可大于可见评论行数（与点赞假数同模式）

**评论交互（v1.2 明确）：**
- 单击 💬 icon → 页面底部弹出评论输入框，用户直接输入发送
- **不支持**点击某条评论行切换为「回复 TA」或嵌套回复他人（与 PRD 6.2.1 一致）
  - **2026-07-12 例外**：允许点击**林小梦回复行**发连续评（落款「我回复 林小梦：」）；用户自己的评论行仍不可点
- 用户可对同一条帖子**多次发评**（平铺追评），发送后评论实时追加到评论列表末尾
- 林小梦回复以「回复 @我」形式展示（见 4.5）

**话题标签着色（v1.6 / 2026-07-12）：**
- 正文内以 **单 `#` 起标**：从 `#` 起着色至空格或常见中英文标点/括号前
- 兼容成对写法 `#话题#`（收尾 `#` 一并纳入着色）
- 仅处理 `content_text` 内匹配段；API `hashtags[]` **不**单独渲染为标签行
- 点击不跳转（v1）

---

### 4.5 评论区展示（v1.2 更新）

**展示规则：**
- 有评论时，评论区紧接在互动栏下方展示
- 全部评论平铺显示，不折叠
- 排序：按发布时间正序（早→晚）
- **评论区为当前用户私有**：仅展示该用户自己发布的评论及对应的 LXM 回复，不展示其他用户的评论内容（与 PRD 6.2.1 一致）；因此同一帖子下，不同用户打开评论区看到的内容互不相同

**「我」展示强制规则（v1.2 新增，与 PRD 6.2.1 一致）：**
- 当前用户自己发布的评论落款**一律显示「我」**
- LXM 回复中的「回复 @XX」引用**一律显示「回复 @我」**
- **禁止**使用 `users.username`、真名、偏好称呼或任何账号 ID 作为评论区展示名

**视觉区分规则：**
- 林小梦的评论：轻度高亮（背景色或小角色 icon 标记），用于快速识别角色回复
- 普通用户的评论：默认样式

## 五、页面级交互逻辑

### 5.1 新内容加载

| 方式 | 说明 |
|---|---|
| 下拉刷新 | 用户手动下拉触发，加载最新动态 |
| 新内容提示条 | 独立 SSE 推送 `feed_new` 事件，页面顶部出现「X 条新动态」提示条，**点击后**滚回顶部并拉取最新列表 |

**技术实现（v1.4 定案，对应 PRD 5.9）：**
- **独立端点** `GET /api/feed/events`（JWT，`text/event-stream`），与对话 `POST /api/chat/send` 的 SSE **完全隔离**，不复用、不混流
- **连接生命周期**：仅朋友圈页打开时 `EventSource` 连接，离开页面断开
- **事件载荷**：`{"type":"feed_new","delta":1}`，前端累加计数；**不传**帖子正文/图片
- **断线**：指数退避重连；重连不补发历史，下拉刷新兜底
- 首页 [New] 徽标仍走 `last_feed_entered_at`（PRD 5.6），与 SSE 提示条分工独立

### 5.2 Feed 分页加载

- 无限滚动：用户上滑至底部触发加载更多
- 每次加载一页数据，数据追加到列表末尾
- 接口：`GET /api/feed/list` **游标分页**；H5 默认每页 **20**（上限 50）
- **首屏 boot 未结束前不挂载底部哨兵**，避免与进页首屏请求抢占（见 TB-LF-010）

### 5.3 首屏加载态（v1.6 更新）

- **进页**：全屏 boot 遮罩 `#feed-boot-loading`（紫色转圈 + 文案「正在进入她的生活…」）
  - 进度驱动遮罩透明度与模糊（对齐首页 loader：越加载越清晰）
  - 门闩：列表返回 + 首屏图就绪（最多 6 张，单图超时约 2.5s）+ 最少展示约 700ms
  - **硬超时 8s** 强制退场；空态/错态同样退场
- **下拉刷新**：仍用**骨架屏（Skeleton Screen）**，不走全屏 boot
- 数据加载完成后内容替换进来

### 5.4 错误态

- 加载失败时，在 Feed 区域内显示提示文案 + 重试按钮
- 同时支持下拉手势重试
- 已加载的内容保持可见，不整页替换为错误态

### 5.5 空态

- 上线前业务侧会预填充动态，正常情况下不出现空态
- 保险起见，空态展示：角色风格插画 + 提示文案（如"小梦还没有发动态哦～"）

## 六、数据来源说明（v1.2 更新）

| 页面元素 | 数据来源 |
|---|---|
| 背景图、头像、昵称、签名 | 角色配置（后台管理） |
| 动态正文、图片 | LIFE002 生成，存储于 `feed_post` 表 |
| 评论内容 | 用户输入 + LIFE004 林小梦回复（**LLM-05**，写入 `feed_comment.lxm_reply`） |
| 点赞状态 | `feed_like` 表（per-user）+ `feed_post.real_likes` 聚合展示数（公式见 PRD 6.1.1） |
| 评论角标 | list `display_comments` = `base_comments × comment_multiplier + 当前用户可见评论条数`（2026-07-12；历史帖默认 0×1） |
| 新动态推送 | 独立 SSE `GET /api/feed/events`，事件 `feed_new`（见 PRD 5.9） |
